GLIC Metal

GlitchCodec/GLICの構造をC++20とMetalへ移植し、映像入力をリアルタイムに破壊・再構成する実験的なオープンソース実装です。原作準拠と高速な視覚近似を、明示的に別の処理モードとして提供します。

プリセットを見る ソースと使い方 互換性レポート

3つの実行モード

01 / STRICT

原作スタイル

Processing互換の乱数順序、四分木分割、予測、量子化、CDF 9/7再構成を優先する37プリセットの高忠実度経路。

02 / FAST MATCH

解析済み高速化

視覚差と処理時間の両方を評価し、安全なプリセットだけに近似とキャッシュを適用。毎フレームの確保を避けます。

03 / COMPAT

全144プリセット

上流プリセットの意味を読み込み、リアルタイム向けの別系統の視覚表現として処理。原作とのピクセル一致は主張しません。

公開仕様

144固定上流プリセット
37原作スタイル対応
960×540最低リアルタイム解像度
20 fps許容性能フロア

30fpsの認証結果は個別のベンチマーク条件とともに記録します。リアルタイム合格を原作との完全一致とは扱いません。

ビルド

サブモジュールを含めて取得し、CMakeでビルドします。MetalとWebカメラアプリにはmacOSとFull Xcodeが必要です。

git clone --recurse-submodules <repository-url>
cd glic-metal
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
ctest --test-dir build --output-on-failure
